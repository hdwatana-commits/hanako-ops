# ファッションハナコOPS Phase 3 実装計画

## テーマ

Phase 3のテーマは「AIが投稿担当者になる」です。運用者は商品探し、コーデ作成、画像生成プロンプト、投稿文、コレクション分類、再投稿判断を毎回手で作らず、AIが作った投稿準備を承認するだけに近づけます。

## 実装方針

- 既存のPhase 1/2を壊さず、上位に `HanakoPhase3Engine` を追加しました。
- 外部AI APIを必須にせず、まずはAI判断・根拠・プロンプト・承認結果を保存できる構造にしています。
- Gemini/ChatGPT/GPT Visionへ渡すプロンプトは生成しますが、APIキーなしでもアプリは動きます。
- AI判断は理由、根拠、参照データを保存します。
- すべてのAI処理は再実行でき、採用・除外・売上・クリックから学習プロフィールを更新します。

## 追加機能

- AI商品評価
- AI画像解析用プロンプト
- AIコーデ提案
- AI画像生成プロンプト
- AI投稿文生成
- AIコレクション提案
- AI再投稿判断
- AI売上分析
- AI改善レポート
- AI投稿スケジュール
- AIダッシュボード
- AI秘書

## 追加ファイル

- `tests/ops-phase3-tests.html`

## 使い方

1. 商品画面で「今朝のおすすめ」を作ります。
2. ホームの「AI担当者に作らせる」を押します。
3. 「今日AIがやること」に売れる商品候補、画像生成待ち、投稿予約、判断待ちが表示されます。
4. 上位プランの「OKして準備」を押すと、投稿パイプライン、ROOM投稿待ち、カレンダーへ入ります。
5. AI秘書に「今日は何を投稿する？」「旅行コーデ作って」「今売れそうなのは？」などを聞けます。

## Supabase

正式テーブルを作る場合は、Phase 1/2 SQL実行後に `supabase-ops-phase3.sql` をSQL Editorで実行します。

## 既知の制約

- 現時点では外部AI APIを自動実行せず、Gemini/ChatGPT/GPT Visionへ渡すプロンプト生成と承認フローを実装しています。
- 本当に画像解析APIを自動実行するには、Edge Function側にAPIキー管理とジョブ実行を追加します。
- AIの売上学習はサンプル数が少ない場合は断定せず、Phase 2の実績補正を参照します。
